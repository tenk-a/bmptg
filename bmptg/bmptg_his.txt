v2.31b	2007-06
・ dmc(8.50bata) でコンパイル通るように修正. (型チェックが厳しい模様)

 問題は libpng 側も要修正...どうも PNG_CONST が有名無実化して const が使われ
  libpng/pngerror.c(116): const static PNG_CONST char png_digit[16] = {
  libpng/pngtrans.c(255): const static PNG_CONST png_byte onebppswaptable[256] = {
  libpng/pngtrans.c(290): const static PNG_CONST png_byte twobppswaptable[256] = {
  libpng/pngtrans.c(325): const static PNG_CONST png_byte fourbppswaptable[256] = {
 のように両方指定があったり....でdmcが怒ってくれるのだった.
 PNG_NO_CONST定義してPNG_CONSTを無効化したら、今度は const でないといけない
 ものが非const化してエラーになるので、結局上記４行の先頭のconstを削除することで対処.
  (dmcでコンパイルする場合は各自が上記４行の修正をしてください)
